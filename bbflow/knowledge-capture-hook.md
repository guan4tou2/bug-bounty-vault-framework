# Knowledge Capture Hook

Automation is useful only if reviewed output improves future decisions.

## Capture Targets

| Target | Use when |
|---|---|
| Pattern | A recurring signal or behavior appears across more than one context. |
| Playbook | A sequence of decisions becomes repeatable. |
| Checklist | A review step can be made mechanical. |
| Reference Card | A compact rule would prevent future mistakes. |

## Required Questions

1. What did this run teach?
2. Is it reusable outside the current target?
3. What should be ignored next time?
4. What evidence stays private?
5. Which note should be updated?

## Safety Rule

Safety baseline: do not copy raw output into the vault. Summarize the learning, link to private evidence only from the private vault, and remove target identifiers from public material.
